# GeoTracker 자동화 고도화 로드맵 (A → B → C)

> 본 문서는 자동화 기반 시계열 지표를 단계적으로 강화하기 위한 **로컬 전용 계획서**입니다.
> 실서버 배포 단계에서는 **B부터가 필수** 입니다. (브라우저 탭에 의존하는 A 수준은 실서비스에 부적합)

---

## 배경

현재 가시성/언급률 등 지표는 사용자가 수동으로 실행한 runs + 자동 스케줄러(브라우저 탭)에서 실행된 runs가 섞여 있다. 날짜별 실행 횟수가 들쭉날쭉해 통계적으로 신뢰하기 어렵다.

**해결 원칙** : "통계/그래프는 일정 간격으로 자동 수집된 표본에서만 계산"
- `auto: true` runs만 지표 계산 대상
- 샘플 부족(< N건/일)인 날은 "데이터 부족" 처리, 평균선에서 제외
- 점진 확장으로 외부 스케줄러 → 일별 롤업 테이블까지 강화

---

## Phase A — 프론트엔드 집계만 (현재 작업)

**목표**: 기존 브라우저 스케줄러가 쌓은 `auto:true` runs를 **바로 집계 시각화**하여 "자동화 고도화됨" 체감을 즉시 제공.

**구현 범위**
- `types.ts`의 `ScrapeRun.auto: boolean` 활용 (이미 추가됨)
- `Automation` 탭 하단에 "**자동화 통계**" 섹션 추가
  - 최근 14~30일 **일별 평균 가시성** 라인차트 (모델별)
  - **일별 브랜드 언급률(%)** 라인차트 (모델별)
  - **모델별 자동 실행 횟수** 바차트 (최근 7일)
  - 일별 표본 수가 N(=3) 미만이면 회색/점선 "데이터 부족" 배지
- 통계 계산은 클라이언트에서 `useMemo`로 수행 (API 추가 없음)

**제약 / 주의**
- 브라우저 탭을 꺼 놓으면 샘플이 안 쌓임 → 사용자 경험에 "탭 열어 두기" 가이드 명시
- IndexedDB 용량 한계(runs 누적)로 장기 지표는 한계 있음 → B/C에서 해결

**완료 기준**
- Automation 탭 통계 섹션에서 자동 수집된 데이터만으로 그래프가 그려진다
- 샘플 수 부족 일자는 시각적으로 구분된다

---

## Phase B — 외부 스케줄러 + 슬롯 정규화 (실서버 배포 시 필수)

**목표**: 브라우저 의존 제거. 일정한 간격(cron slot)으로 서버가 강제 실행 → 표본 간격이 **항상 균등**.

**구현 범위**
1. **Vercel Cron 또는 GitHub Actions 주기 호출**
   - `/api/cron/scheduled-batch` 신규 엔드포인트 생성
   - Secret 헤더(`X-Cron-Secret`)로 무단 호출 방지
   - Vercel 옵션: `vercel.json`의 `crons` 배열로 매 시간/6시간 호출
   - GitHub Actions 옵션: 기존 `deployment-*.yml` 옆에 `scheduled-run.yml` 추가 (workflow_dispatch + cron)
2. **슬롯 해시**: 실행마다 `intervalSlot = YYYY-MM-DDTHH` (또는 6시간 단위 버킷) 기록
   - 동일 슬롯에서 중복 실행 방지 (idempotency)
   - `ScrapeRun`에 `intervalSlot?: string` 필드 추가
3. **서버 측 저장소**: 현재 IndexedDB(브라우저)에만 저장 → 서버에서 실행 시 결과를 어디에 저장할지 결정
   - 옵션 B-1: Vercel KV / Upstash Redis (간단, 빠름)
   - 옵션 B-2: Postgres (Supabase/Neon) (SQL 집계 가능)
   - **권장**: B-2 (Phase C에서 롤업 테이블 활용)

**완료 기준**
- 서버 측에서 정해진 간격으로 자동 실행되어 runs가 쌓인다
- 동일 슬롯 중복 실행이 차단된다
- 일별 표본 수가 거의 균등하게 유지된다 (운영 기간 중 > 90%)

**추정 공수**: 2~3일

---

## Phase C — 일별 롤업 테이블 + 이상치 탐지 (고급)

**목표**: runs 원본이 쌓이면 쿼리 비용이 커짐 → 매일 자정 집계된 일별 스냅샷으로 가벼운 시계열 분석.

**구현 범위**
1. **일별 롤업 테이블** (`daily_stats`)
   - PK: `(date, provider, prompt_hash)` 또는 `(date, provider)` 수준 롤업
   - 컬럼: `avg_visibility`, `mention_rate`, `citation_rate`, `sample_count`, `std_dev`
   - 매일 KST 00:10 cron으로 전일 분 집계 → 롤업 테이블에 INSERT
2. **경쟁사/키워드별 롤업**
   - `prompt_tag` 단위로 묶어 카테고리별 트렌드
3. **이상치 탐지**
   - 7일 이동평균 대비 2σ 벗어나면 "이상치" 표식 + 드리프트 알림 강화
   - Z-score 기반
4. **장기 시계열 뷰**
   - Automation 탭 또는 Home 탭: 최근 90일/180일 트렌드 (원본 runs 대신 롤업 조회)

**완료 기준**
- 180일 이상 장기 그래프가 빠르게 렌더된다 (롤업 쿼리 사용)
- 이상치 자동 플래그가 UI에 표시된다
- IndexedDB를 롤업 이후 30일치로 축소해도 데이터 손실 없음

**추정 공수**: 4~5일

## 회귀 방지 체크리스트

- [ ] `auto: true` 플래그가 스케줄 실행에만 부여되는지 확인 (`runScheduledBatch`에서만 세팅)
- [ ] 수동 실행이 통계 집계에 섞이지 않는지 Automation 통계 섹션에서 재검증
- [ ] Phase B 이후: `intervalSlot` 중복 실행이 실제 차단되는지 통합 테스트
- [ ] Phase C 이후: 롤업 테이블이 원본 runs와 동일한 결과를 산출하는지 샘플 쿼리로 검증
